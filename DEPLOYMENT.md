# نشر HalalStream أونلاين

هذا المشروع أصبح جاهزاً للنشر كخدمة Docker واحدة: الواجهة وواجهة البرمجة ومحرك إزالة المعازف يعملون من نفس خادم FastAPI.

## المتطلبات المناسبة

- خادم يدعم Docker.
- ذاكرة لا تقل عن 4GB، والأفضل 8GB إذا كانت المقاطع طويلة.
- قرص دائم لا يقل عن 20GB لحفظ الملفات المؤقتة والنتائج.
- وقت معالجة كاف؛ عزل الصوت قد يأخذ عدة دقائق حسب طول المقطع وقوة الخادم.

الخدمات القصيرة من نوع serverless ليست مناسبة غالباً، لأن تنزيل المقطع وتشغيل نموذج العزل يحتاجان وقتاً وذاكرة وملفات مؤقتة.

## الخيار المجاني الأسهل: Hugging Face Spaces

هذا هو الخيار الأنسب كبداية مجانية لهذا المشروع، لأننا نحتاج Docker وPython وFFmpeg ونموذج عزل صوتي.

الخطوات:

1. افتح حساباً في Hugging Face.
2. اضغط `New Space`.
3. اختر الاسم مثل `halalstream`.
4. اختر `Docker` في خانة SDK.
5. اختر `Public` للتجربة أو `Private` إن أردت إخفاءه.
6. ارفع ملفات المشروع أو اربط مستودع GitHub.
7. انتظر البناء. بعد اكتماله افتح رابط الـ Space وسيعمل الموقع من نفس الخادم.

الملف `README.md` يحتوي إعدادات Hugging Face Spaces:

```yaml
sdk: docker
app_port: 8000
```

ملاحظات:

- النسخة المجانية مناسبة للتجربة والعرض الأول، لكنها قد تكون بطيئة مع المقاطع الطويلة.
- لا تعتمد عليها كمنصة ضغط عالٍ لعدد كبير من المستخدمين.
- إذا أعاد Space التشغيل فقد تضيع ملفات المهام المؤقتة، وهذا مقبول لأن الملفات الناتجة يجب تحميلها بعد اكتمال المعالجة.

## الخيار الأرخص مع Modal

إذا أردت تخفيض تكلفة Hugging Face، شغل الموقع على `CPU basic` واترك التنقية لعامل Modal. في هذا الوضع:

- التحميل المباشر يبقى سريعاً ولا يستخدم Modal.
- الفحص والتنقية فقط يذهبان إلى Modal GPU عند الطلب.
- لا يحتاج الموقع إلى GPU دائم على Hugging Face.

راجع `MODAL.md` لإعداد الأسرار ونشر `modal_worker.py`.

## لماذا ليس Netlify؟

Netlify مناسب جداً للواجهات الثابتة، لكنه ليس مناسباً لتشغيل هذا المشروع كاملاً، لأن التطبيق يحتاج:

- خادم Python مستمر.
- FFmpeg.
- نموذج Demucs.
- تنزيل ملفات كبيرة نسبياً.
- معالجة قد تستغرق عدة دقائق.

يمكن لاحقاً وضع الواجهة على Netlify والخادم على Hugging Face أو VPS، لكن كبداية أبسط وأقل مشاكل: شغّل الكل في Hugging Face Spaces.

## اختبار Docker محلياً

من مجلد المشروع:

```bash
docker build -t halalstream .
docker run --rm -p 8000:8000 -v halalstream-data:/home/user/app/storage halalstream
```

ثم افتح:

```text
http://127.0.0.1:8000
```

## النشر على VPS

على الخادم:

```bash
git clone YOUR_REPO_URL halalstream
cd halalstream
docker build -t halalstream .
docker run -d \
  --name halalstream \
  --restart unless-stopped \
  -p 8000:8000 \
  -v halalstream-data:/home/user/app/storage \
  -e HALALSTREAM_STORAGE_DIR=/home/user/app/storage \
  -e HALALSTREAM_DEMUCS_JOBS=1 \
  halalstream
```

بعدها اربط النطاق عبر Nginx أو Cloudflare Tunnel أو لوحة مزود الاستضافة.

## النشر على Render

الملف `render.yaml` موجود كنقطة بداية. اربط المستودع في Render واختر Blueprint/Infrastructure as Code، وسيستخدم:

- `Dockerfile`
- فحص الصحة: `/api/health`
- قرص دائم على `/home/user/app/storage`

قد تحتاج خطة بموارد أعلى إذا كانت المقاطع طويلة أو كان هناك أكثر من مستخدم في نفس الوقت.

## ملاحظات مهمة قبل الإطلاق

- لا ترفع مجلد `storage/jobs` إلى المستودع؛ هذا مجلد نتائج ومؤقتات.
- بعض المنصات قد تمنع روابط معينة أو تحتاج Cookies حسب مصدر الفيديو وسياسات الموقع.
- أبقِ `HALALSTREAM_DEMUCS_JOBS=1` في البداية لأنه الأكثر ثباتاً. ارفعه لاحقاً فقط إذا كان الخادم قوياً وتم اختبار النتائج.
- عند الإطلاق العام، أضف حدود حجم الملفات وعدد المهام المتزامنة حتى يبقى الخادم مستقراً.
- إذا كان لديك GitHub Student Developer Pack، فراجع `STUDENT_HOSTING.md` لأن VPS طالب هو الخيار الأفضل لروابط YouTube.
