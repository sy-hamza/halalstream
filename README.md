# HalalStream / إزالة المعازف

تطبيق لتنزيل المقاطع وفحص الصوت وإزالة المعازف قبل السماح بالتحميل، يعمل محلياً للتجربة ويمكن نشره أونلاين عبر Docker.

## التشغيل المحلي

من داخل مجلد المشروع شغّل:

```powershell
.\run-local.bat
```

ثم افتح:

```text
http://127.0.0.1:8000
```

لا تفتح `index.html` مباشرة إذا أردت معالجة حقيقية، لأن المتصفح وحده لا يستطيع تنزيل فيديوهات YouTube ولا تشغيل محرك العزل.

## كيف يعمل

1. الواجهة ترسل الرابط أو الملف إلى خادم المعالجة.
2. الخادم يستخدم `yt-dlp` لتنزيل روابط الفيديو.
3. الخادم يستخدم `ffmpeg` لاستخراج الصوت.
4. الخادم يستخدم محرك عزل صوتي لعزل الصوت البشري عن الطبقة غير الصوتية.
5. إذا لم تظهر معازف يسمح بتنزيل الملف الأصلي.
6. إذا ظهرت معازف يوقف التحميل، ويعرض الموعظة، ولا يجهز الملف إلا بعد الضغط على زر إزالة المعازف.
7. بعد الموافقة يجهز الخادم خيارين: صوت منقّى فقط، أو مقطع كامل بالصوت المنقّى.

## الملفات الناتجة

تخزن المهام داخل:

```text
storage/jobs
```

كل مهمة لها مجلد مستقل يحتوي الملف الأصلي، الصوت المستخرج، ملفات العزل، والملف المنقّى عند اكتماله.

## ملاحظات مهمة

- أول تشغيل لمحرك العزل قد يأخذ وقتاً أطول لأنه يجهز النموذج.
- يجهز الخادم الصوت المنقّى مرة واحدة ثم يستخدمه لخيار الصوت وخيار الفيديو، وهذا يقلل وقت مرحلة التسليم.
- جودة الحذف تعتمد على المقطع نفسه. في بعض المقاطع قد يبقى أثر خفيف أو يتأثر الصوت البشري.
- روابط YouTube تتغير كثيراً. يستخدم الخادم عميل Android داخل `yt-dlp` لأنه نجح مع الروابط التي ترفض عميل الويب بخطأ 403.
- يمكن تجربة تعدد أنوية محرك العزل من المتغير `HALALSTREAM_DEMUCS_JOBS`، لكنه مضبوط افتراضياً على `1` لأنه الأكثر ثباتاً على Windows.
- للنشر أونلاين راجع `DEPLOYMENT.md`.
- لتشغيل التنقية على Modal GPU عند الطلب بدل دفع GPU دائم على Hugging Face راجع `MODAL.md`.
- إذا كان لديك GitHub Student Developer Pack فراجع `STUDENT_HOSTING.md` لتشغيله على VPS مجاني/مدعوم.
